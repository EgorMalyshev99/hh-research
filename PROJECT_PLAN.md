**Качество и надёжность**

- Модульные / интеграционные тесты API (`apps/api`): auth, settings, search run (моки hh/LLM), Drizzle на testcontainers или sqlite
- Фронт: unit-тесты утилит и composables; при необходимости e2e (Playwright) — логин, запуск поиска (мок API)
- CI: `lint` + `check-types` + `build` (+ тесты, когда появятся); кэш pnpm/turbo
- Единый контракт ошибок API (код, сообщение, `requestId`) и отображение в UI (toast / inline), в т.ч. LLM timeout, rate limit, сеть

**Инфраструктура и эксплуатация**

- Наблюдаемость: структурированные логи уже есть — добавить корреляцию `requestId`, опционально OpenTelemetry / экспорт в систему по выбору
- Ошибки фронта: Sentry или аналог (только с явным согласием на внешний сервис)
- Docker-образы для api + web (multi-stage), `docker compose` профиль «всё приложение» для локального прод-подобного запуска
- Версионирование миграций Drizzle (`drizzle-kit generate`) вместо только `push` в проде

**Продукт и UX**

- Главная: фильтры/сортировка списка вакансий, пагинация при росте данных, детальная карточка / drawer по клику
- Настройки: валидация и UX сложных полей (schedule, area), подсказки по полям hh API
- История: экспорт CSV, фильтр по статусу/дате; графики — больше метрик из `search_runs` / `stats`
- «Сохранённые поиски» или шаблоны запросов без повторного ввода
- Уведомления о завершении длинного поиска (in-app / браузерные / email — на выбор)

**Безопасность и данные**

- Аудит зависимостей (`pnpm audit`), политика обновлений; секреты только в env, никогда в репо
- Ограничение размера тел запросов, защита от злоупотребления SSE (длина сессии, лимиты на пользователя)
- Резервное копирование PostgreSQL (скрипт / дока), политика хранения `search_runs` и логов

**Техдолг и соглашения**

- Полная миграция `eslint-plugin-boundaries` на объектные селекторы без legacy-предупреждений (если останутся)
- Выравнивание дизайна: токены shadcn vs кастомный `@theme` (indigo/violet), единая палитра
- Документация для контрибьюторов: `CONTRIBUTING.md`, шаблон PR, как запускать только api или только web

**Амбициозно / вне v1**

- Мультипользовательский деплой, роли, приглашения
- OAuth hh.ru для расширенных сценариев API (если понадобится за пределами публичного поиска)
